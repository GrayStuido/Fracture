How to make a Fracture Instance

Introduction

This tutorial will guide you through setting up GitHub and Bitwarden, two essential tools for modern web development and online security. We'll then use these tools to fork a repository and set up a GitHub Pages website.

GitHub: A platform for version control and collaboration. It lets you and others work together on projects from anywhere.
Bitwarden: A secure and free password manager that stores encrypted passwords online.

Step 1: Create a GitHub Account

  1. Go to GitHub's website.
  2. Click on "Sign up" in the top-right corner.
  3. Enter your email address, create a password, and choose a username.
  4. Complete the verification process.

Your GitHub username will be visible in your project URLs, so make sure its not a blocked keyword

Step 2: Create a Bitwarden Account

  1. Visit Bitwarden's website.
  2. Click on "Create Account" in the top-right corner.
  3. Enter your email address and create a strong master password.
  4. Complete the registration process.

Your master password should be a new password. Make it strong and memorable, as Bitwarden cannot recover it if you forget

Step 3: Install Bitwarden Extension

  1. Go to the Bitwarden Chrome Extension page.
  2. Click "Add to Chrome" (or your browser's equivalent).
  3. Confirm the installation when prompted.
  4. Once installed, click on the Bitwarden icon in your browser toolbar.
  5. Log in with your Bitwarden account credentials.

Step 4: Set Up a Passkey for GitHub with Bitwarden

  1. Ensure you're logged into the Bitwarden extension.
  2. Go to GitHub and log out if you're currently logged in.
  3. Log back into GitHub.
  4. You should be prompted to add a passkey with Bitwarden. Follow the prompts to set it up.

If you're not prompted, you can manually add a passkey by following GitHub's guide on adding a passkey.

Using a passkey allows for more secure and convenient logins without needing your phone for two-factor authentication.

Step 5: Fork the Fracture Repository

  1. Go to the Fracture GitHub repository.
  2. Click the "Fork" button in the top-right corner of the page.
  3. Choose where to fork the repository (your personal account).
  4. Wait for GitHub to create a copy of the repository under your account.
Forking: Creating a personal copy of someone else's project. It allows you to freely experiment with changes without affecting the original project.

Step 6: Set Up GitHub Pages

  1. Go to your forked repository's page on GitHub.
  2. Click on the "Settings" tab near the top of the page.
  3. In the left sidebar, click on "Pages".
  4. Under "Build and deployment", set "Source" to "Deploy from a branch".
  5. Select "main" as the branch.
  6. Click "Save".
GitHub Pages: A free hosting service provided by GitHub that allows you to publish your website directly from your GitHub repository.

Step 7: Check Your Deployment

  1. Go to the "Actions" tab in your repository.
  2. You should see a workflow running to deploy your site.
  3. Wait for 2-3 minutes until you see a green checkmark, indicating successful deployment.
  4. Click on the completed workflow to find the link to your deployed page.

Your site will be available at https://[your-username].github.io/[whatever-you-named-it]

Conclusion

Congratulations! You've now:

  • Set up GitHub and Bitwarden accounts
  • Installed the Bitwarden browser extension
  • Created a secure passkey for GitHub
  • Forked a repository
  • Deployed a website using GitHub Pages

These skills form a great foundation for further web development projects and maintaining good online security practices.
